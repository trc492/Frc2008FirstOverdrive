FunctionsDescription
_do_cinit This function copies the ROM to memory blocks according to the _ciniit table.
_entry This is the main entry point. After the processor is reset, it jumps to the code located at RESET_VECTOR which is where this code is located. This start up location contains only one instruction. It immediately jumps to the _startup function which is located in the _startup code section.
_startup This is the startup function. It initializes the processor environment including setting up the stack and processor flags. Then it goes into a loop clearing memory, copying ROM to memory blocks and executing user's main code.
_user_putc This function overrides the putc() interface to C18 2.4 output stream functions and writes a byte to the serial port's transmit queue.

InputCalibrate This function initializes for joystick calibration.
InputInitialize This function initializes the input data. This includes calibrating all in-use analog channels. The Operator Interface has 4 joystick ports and each has 4 analog channels totalling 16 analog channels. This function allows specifying which joystick ports and which analog channels are in used so that only those channels are initialized and calibrated.
InputPrintData This function prints the input data to the debug console.
InputProcessData This function processes the input data from the Operator Interface. If we are in calibration mode, it will perform the calibration cycle. Otherwise, it will adjust the input data according to the calibration data.
